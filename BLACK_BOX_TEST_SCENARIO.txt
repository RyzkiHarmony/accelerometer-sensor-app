================================================================================
SKENARIO PENGUJIAN BLACK BOX - ROAD DAMAGE DETECTOR
================================================================================
Tanggal Dokumen : 21 Januari 2026
Aplikasi        : Road Damage Detector (Android)
Tujuan          : Memverifikasi fungsionalitas utama aplikasi dari perspektif pengguna akhir.

PETUNJUK PENGISIAN:
- Hasil: Isi dengan "PASS" jika sesuai harapan, atau "FAIL" jika tidak.
- Catatan: Tuliskan detail bug atau perilaku aneh jika status "FAIL".

================================================================================
MODUL 1: HOME SCREEN & PEREKAMAN (RECORDING)
================================================================================

[TC-001] Tampilan Awal (Idle State)
Langkah:
1. Buka aplikasi.
2. Pastikan berada di tab "Home" (ikon rumah).
Ekspektasi:
- Tombol "Start Recording" terlihat jelas di tengah/bawah.
- Status "Ready to start" atau indikator sensor terlihat.
- Header aplikasi tidak tertutup status bar (jam/baterai HP).

[TC-002] Memulai Perekaman
Langkah:
1. Tekan tombol "Start Recording".
2. Berikan izin lokasi/penyimpanan jika diminta.
Ekspektasi:
- Tampilan berubah menjadi mode "Active Session".
- Timer durasi mulai berjalan.
- Jarak (Distance) mulai terhitung (jika bergerak).
- Peta mini (jika ada) menampilkan lokasi saat ini.

[TC-003] Deteksi Guncangan (Simulasi Jalan Rusak)
Langkah:
1. Saat perekaman aktif, goyangkan HP secara vertikal (simulasi melewati lubang).
Ekspektasi:
- Grafik G-Force menunjukkan lonjakan.
- Counter "Potholes" atau "Events" bertambah (jika ditampilkan di UI recording).
- Tidak ada crash pada aplikasi saat guncangan keras.

[TC-004] Menghentikan Perekaman
Langkah:
1. Tekan tombol "Stop" atau ikon kotak merah.
Ekspektasi:
- Perekaman berhenti.
- Data perjalanan disimpan.
- Navigasi otomatis kembali ke menu awal atau ringkasan.
- Muncul notifikasi/toast bahwa trip berhasil disimpan.

================================================================================
MODUL 2: RIWAYAT PERJALANAN (TRIP LIST)
================================================================================

[TC-005] Menampilkan Daftar Riwayat
Langkah:
1. Pindah ke tab "History" atau "List" (ikon daftar).
Ekspektasi:
- Daftar perjalanan yang baru direkam muncul paling atas (default).
- Informasi tanggal, jarak, dan durasi tampil pada setiap item.

[TC-006] Fitur Sorting (Pengurutan)
Langkah:
1. Di halaman List, tekan ikon "Sort" (biasanya di pojok kanan atas).
2. Pilih opsi "Terlama" atau "Jarak Terjauh".
Ekspektasi:
- Urutan daftar berubah sesuai opsi yang dipilih.
- Ikon sort berfungsi responsif.

[TC-007] Navigasi ke Detail
Langkah:
1. Klik salah satu item perjalanan di daftar.
Ekspektasi:
- Aplikasi berpindah ke halaman "Trip Detail".
- Data yang ditampilkan sesuai dengan item yang diklik (cek ID atau Jam).

================================================================================
MODUL 3: DETAIL PERJALANAN & EKSPOR DATA
================================================================================

[TC-008] Tampilan Detail Perjalanan
Langkah:
1. Buka salah satu detail perjalanan.
Ekspektasi:
- Peta menampilkan rute perjalanan (garis polyline).
- Marker atau indikator titik kerusakan terlihat di peta.
- Statistik (Jarak, Durasi) sesuai dengan data di daftar.
- Grafik G-Force tampil di bagian bawah.
- Header tidak tertutup status bar.

[TC-009] Tombol Kembali (Back Navigation)
Langkah:
1. Tekan tombol ikon "Home/Back" di pojok kiri atas halaman detail.
Ekspektasi:
- Kembali ke halaman Trip List (atau Home, tergantung alur).
- Tidak keluar dari aplikasi.

[TC-010] Fitur Download CSV/JSON (Verifikasi Fitur Baru)
Langkah:
1. Di halaman detail, tekan tombol ikon "Download" (panah ke bawah).
Ekspektasi:
- Muncul notifikasi (Snackbar/Toast): "Tersimpan di Download/RoadDamageDetector".
- Cek File Manager HP di folder `Download/RoadDamageDetector`.
- Terdapat file `.csv` (data sensor) dan `.json` (metadata) dengan nama sesuai ID trip.
- Isi file CSV tidak kosong (bisa dibuka di viewer/Excel).

================================================================================
MODUL 4: PENGATURAN (SETTINGS)
================================================================================

[TC-011] Mengubah Sensitivitas
Langkah:
1. Masuk ke menu Settings.
2. Ubah nilai Threshold (misal dari 2.0 ke 4.0).
3. Kembali ke Home dan coba rekam.
Ekspektasi:
- Nilai tersimpan.
- Dibutuhkan guncangan lebih keras untuk memicu deteksi (jika logika threshold aktif).

[TC-012] Informasi Versi Aplikasi
Langkah:
1. Cek bagian bawah menu Settings.
Ekspektasi:
- Menampilkan versi aplikasi yang benar.
